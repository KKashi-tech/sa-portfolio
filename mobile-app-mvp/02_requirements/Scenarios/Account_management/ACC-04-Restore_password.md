# Сценарий 10: Восстановление пароля

**Когда:** я забываю пароль и хочу восстановить его по электронной почте.  

**Примечания:**  
- Письмо только для существующих аккаунтов  
- Тексты письма согласовать с UX/UI  
- Требования к безопасности уточнить позже  

**Критерии приёмки:**  
- Пользователь может восстановить пароль по электронной почте  
- Сообщение при несуществующей почте  
- После сброса вход только с новым паролем  

**Сценарий 1: Успешное восстановление**  
- **Дано:** зарегистрированный, неавторизованный пользователь на экране входа  
- **Когда:** выбирает «Забыли пароль?» → вводит корректный email → подтверждает  
- **Тогда:**  
  - Пользователь получает письмо с новым паролем  
  - Сообщение:  
    ```
    Старый пароль сброшен, новый пришел на вашу почту
    ```

**Сценарий 2: Email не зарегистрирован**  
- **Дано:** пользователь на экране восстановления пароля  
- **Когда:** вводит незарегистрированный email → подтверждает  
- **Тогда:**  
Пользователь с такой почтой не найден.
- Письмо не отправляется  
- Можно изменить адрес на этом экране

**Сценарий 3: Вход только с новым паролем**  
- **Дано:** пользователь получил новый пароль  
- **Когда:** вводит email и новый пароль  
- **Тогда:** вход выполняется
