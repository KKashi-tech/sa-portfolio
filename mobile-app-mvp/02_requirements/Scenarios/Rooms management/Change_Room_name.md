# Сценарий 7: Переименование комнаты

**Когда:** как пользователь, я хочу менять название комнаты, чтобы исправлять ошибки или переименовывать комнату.  

**Примечания:** название комнаты должно быть уникальным в пределах дома.  

**Критерии приёмки:**  
- Авторизованный пользователь может изменить название комнаты  
- Ошибка при пустом поле или дублирующемся названии  

**Сценарий 1: Успешное переименование**  
- **Дано:** пользователь находится в приложении, видит список комнат  
- **Когда:** выбирает комнату → «Переименовать» → вводит новое название → подтверждает  
- **Тогда:**  
  - Название обновлено  
  - Сообщение:  
    ```
    Название комнаты изменено.
    ```

**Сценарий 2: Ошибка — название уже используется**  
- **Дано:** в доме есть другие комнаты  
- **Когда:** вводит уже существующее название → подтверждает  
- **Тогда:**  
  - Сообщение:  
    ```
    Комната с таким названием уже существует.
    ```  
  - Название не меняется

**Сценарий 3: Ошибка — пустое поле**  
- **Дано:** экран переименования для комнаты `<старое название>`  
- **Когда:** поле пустое → подтверждает  
- **Тогда:**  
  - Сообщение:  
    ```
    Поле пустое — введите название комнаты.
    ```  
  - Название не меняется
