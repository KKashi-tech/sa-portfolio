## Добавление лампы Stets по уникальному коду вручную

**Когда я не могу отсканировать QR-код**,  
я хочу иметь возможность добавить лампу Stets по уникальному коду вручную,  
чтобы не связываться с поддержкой для добавления устройства.

### Примечание
- В уникальном коде 12 символов.
- Необходимо уточнить у стейкхолдеров, какие символы могут использоваться.

### Критерии приёмки
- Авторизованный пользователь может добавить лампу Stets по уникальному коду вручную.
- Если пользователь вводит слишком короткий или слишком длинный код — отображается подсказка.
- Если уникальный код недействителен, отображается сообщение об ошибке.
- Если устройство уже добавлено, отображается сообщение об ошибке.
- При попытке добавить неподдерживаемое устройство отображается отдельная ошибка.

### Сценарий 1: Авторизованный пользователь может добавить лампу Stets по уникальному коду вручную

**Дано**  
авторизованный пользователь находится в приложении

**Когда**
- пользователь выбирает опцию «Добавить устройство»
- выбирает способ «Добавить вручную»
- вводит корректный `<уникальный код>` лампы Stets

**Тогда**
- устройство успешно добавляется в систему
- система получает его `<уникальный код>`, `<название модели>` и `<тип устройства>`
- пользователь видит сообщение:  
  Лампа успешно добавлена.
- лампа отображается в списке устройств

#### Примеры

| Тип устройства | Название         | Уникальный код |
|---------------|------------------|---------------|
| Лампа Stets   | LampStetsLP15    | MD12748920HT  |
| Лампа Stets   | LampLP12 Plus    | LP4759375LFN  |

### Сценарий 2: Если пользователь вводит слишком короткий или слишком длинный код — отображается подсказка

**Дано**  
авторизованный пользователь находится в приложении

**Когда**
- пользователь выбирает опцию «Добавить устройство»
- выбирает способ «Добавить вручную»
- вводит слишком короткий или слишком длинный уникальный код лампы Stets

**Тогда**
- пользователь видит подсказку:  
  Код должен содержать 12 символов - проверьте введенные данные
- пользователь не может продолжить добавление устройства

### Сценарий 3: Если уникальный код недействителен, отображается сообщение об ошибке

**Дано**  
авторизованный пользователь находится на экране добавления устройства

**Когда**
- пользователь вводит `<недействительный>` или неверно отформатированный уникальный код

**Тогда**
- пользователь видит сообщение:  
  Не удалось добавить устройство: уникальный код недействителен.
- устройство не добавляется в список

### Сценарий 4: Если устройство уже добавлено, отображается сообщение об ошибке

**Дано**  
авторизованный пользователь находится на экране добавления устройства

**Когда**
- пользователь вводит уникальный код, который уже используется в приложении

**Тогда**
- пользователь видит сообщение:  
  Не удалось добавить устройство: такое устройство уже есть в приложении.
- устройство не добавляется в список

### Сценарий 5: При попытке добавить неподдерживаемое устройство отображается отдельная ошибка

**Дано**  
авторизованный пользователь находится на экране добавления устройства

**Когда**
- пользователь вводит уникальный код `<неподдерживаемого устройства>`

**Тогда**
- пользователь видит сообщение:  
  Данное устройство не поддерживается в текущей версии приложения.
- устройство не добавляется в список

**Примеры неподдерживаемых устройств**
- Розетка Stets
- Любое устройство другого производителя
